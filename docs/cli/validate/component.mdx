# cldctl validate component

Validate a component configuration file without deploying.

<Note>
Use `cldctl validate comp` as shorthand for `cldctl validate component`.
</Note>

## Synopsis

```bash
cldctl validate component [path] [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `[path]` | Path to the component directory or cld.yml file (default: current directory) |

## Options

| Option | Description |
|--------|-------------|
| `-f, --file <path>` | Path to cld.yml if not in default location |

## Examples

```bash
# Validate current directory
cldctl validate component

# Using alias
cldctl validate comp

# Validate specific directory
cldctl validate component ./my-app

# Validate specific file
cldctl validate component -f custom-component.yml
```

## Output

```
$ cldctl validate component ./my-app

Component configuration is valid!
```

**On validation error:**

```
$ cldctl validate component ./invalid-app

Error: validation failed: [VALIDATION_ERROR] database type "invalid" not supported
  supported types: postgres, mysql, mongodb, redis
```
