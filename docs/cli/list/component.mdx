# arcctl list component

List components available locally or deployed to an environment.

<Note>
Use `arcctl ls comp` or `arcctl list comp` as shorthand.
</Note>

## Synopsis

```bash
arcctl list component [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-e, --environment <name>` | Target environment (lists deployed components) |
| `-o, --output <format>` | Output format: `table`, `json`, `yaml` |
| `--backend <type>` | State backend type |
| `--backend-config <key=value>` | Backend configuration |

## Behavior

- **Without `-e`**: Lists all locally built or pulled components (similar to `docker images`)
- **With `-e`**: Lists all components deployed to the specified environment

## Examples

```bash
# List local components
arcctl list component

# Using alias
arcctl ls comp

# List deployed components
arcctl list component -e production

# Output as JSON
arcctl list component -e staging -o json

# Output as YAML
arcctl list component -e staging -o yaml
```

## Output

**Local components:**

```
$ arcctl list component

docker.io/library/nginx           1.25       pulled      42MB     3 days ago
```

**Deployed components:**

```

Datacenter:  aws-production
```
