# CLI Overview

cldctl provides a comprehensive CLI for building, pushing, and deploying components and datacenters.

## Installation

```bash
# Install cldctl
go install github.com/davidthor/cldctl@latest

# Verify installation
cldctl --version
```

## Command Structure

cldctl uses an intuitive action-first command structure:

```
cldctl <action> <resource> [arguments] [flags]
```

### Global Flags

| Flag | Description |
|------|-------------|
| `--backend <type>` | State backend type (`local`, `s3`, `gcs`, `azurerm`) |
| `--backend-config <key=value>` | Backend-specific configuration (repeatable) |
| `--help, -h` | Show help for command |
| `--version` | Show version information |

## Command Aliases

For convenience, cldctl supports shorthand aliases:

| Full Command | Alias |
|--------------|-------|
| `cldctl build component` | `cldctl build comp` |
| `cldctl deploy datacenter` | `cldctl deploy dc` |
| `cldctl list environment` | `cldctl list env` or `cldctl ls env` |

```bash
# These are equivalent:
cldctl deploy component myapp:latest -e staging
cldctl deploy comp myapp:latest -e staging

# These are equivalent:
cldctl build datacenter . -t ghcr.io/myorg/dc:v1.0.0
cldctl build dc . -t ghcr.io/myorg/dc:v1.0.0

# These are equivalent:
cldctl list environment
cldctl ls env
```

## State Backends

cldctl supports multiple state backends for storing deployment state:

### Local Backend (Default)

```bash
cldctl deploy component myapp:latest -e staging
```

State is stored in `~/.cldctl/state/`.

### S3 Backend

```bash
cldctl deploy component myapp:latest -e staging \
  --backend s3 \
  --backend-config bucket=my-cldctl-state \
  --backend-config region=us-east-1 \
  --backend-config key=cldctl
```

### GCS Backend

```bash
cldctl deploy component myapp:latest -e staging \
  --backend gcs \
  --backend-config bucket=my-cldctl-state \
  --backend-config prefix=cldctl
```

### Azure Backend

```bash
cldctl deploy component myapp:latest -e staging \
  --backend azurerm \
  --backend-config storage_account_name=myaccount \
  --backend-config container_name=cldctl-state \
  --backend-config key=cldctl
```

### Environment Variables

Backend configuration can also be provided via environment variables:

```bash
export CLDCTL_BACKEND=s3
export CLDCTL_BACKEND_S3_BUCKET=my-cldctl-state
export CLDCTL_BACKEND_S3_REGION=us-east-1

cldctl deploy component myapp:latest -e staging
```

## Output Formats

Many commands support different output formats:

```bash
# Default table format
cldctl list component -e staging

# JSON output (for scripting)
cldctl list component -e staging --output json

# YAML output
cldctl list component -e staging --output yaml
```
