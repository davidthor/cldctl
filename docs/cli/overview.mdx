# CLI Overview

arcctl provides a comprehensive CLI for building, pushing, and deploying components and datacenters.

## Installation

```bash
# Install arcctl
go install github.com/architect-io/arcctl@latest

# Verify installation
arcctl --version
```

## Command Structure

```
arcctl [command] [subcommand] [arguments] [flags]
```

### Global Flags

| Flag | Description |
|------|-------------|
| `--backend <type>` | State backend type (`local`, `s3`, `gcs`, `azurerm`) |
| `--backend-config <key=value>` | Backend-specific configuration (repeatable) |
| `--help, -h` | Show help for command |
| `--version` | Show version information |

## Command Aliases

For convenience, arcctl supports shorthand aliases:

| Full Command | Alias |
|--------------|-------|
| `arcctl component deploy` | `arcctl deploy` |
| `arcctl datacenter` | `arcctl dc` |
| `arcctl environment` | `arcctl env` |
| `arcctl component` | `arcctl comp` |

```bash
# These are equivalent:
arcctl component deploy ./my-app -e staging
arcctl deploy ./my-app -e staging

# These are equivalent:
arcctl datacenter build . -t ghcr.io/myorg/dc:v1.0.0
arcctl dc build . -t ghcr.io/myorg/dc:v1.0.0

# These are equivalent:
arcctl environment list
arcctl env list
```

## State Backends

arcctl supports multiple state backends for storing deployment state:

### Local Backend (Default)

```bash
arcctl deploy ./my-app -e staging
```

State is stored in `~/.arcctl/state/`.

### S3 Backend

```bash
arcctl deploy ./my-app -e staging \
  --backend s3 \
  --backend-config bucket=my-arcctl-state \
  --backend-config region=us-east-1 \
  --backend-config key=arcctl
```

### GCS Backend

```bash
arcctl deploy ./my-app -e staging \
  --backend gcs \
  --backend-config bucket=my-arcctl-state \
  --backend-config prefix=arcctl
```

### Azure Backend

```bash
arcctl deploy ./my-app -e staging \
  --backend azurerm \
  --backend-config storage_account_name=myaccount \
  --backend-config container_name=arcctl-state \
  --backend-config key=arcctl
```

### Environment Variables

Backend configuration can also be provided via environment variables:

```bash
export ARCCTL_BACKEND=s3
export ARCCTL_BACKEND_S3_BUCKET=my-arcctl-state
export ARCCTL_BACKEND_S3_REGION=us-east-1

arcctl deploy ./my-app -e staging
```

## Output Formats

Many commands support different output formats:

```bash
# Default table format
arcctl component list -e staging

# JSON output (for scripting)
arcctl component list -e staging --output json

# YAML output
arcctl component list -e staging --output yaml
```

| Code | Description |
|------|-------------|
| `0` | Success |
| `1` | General error |
| `2` | Invalid arguments |
| `3` | State locked |
| `4` | Validation error |
| `5` | Resource not found |
